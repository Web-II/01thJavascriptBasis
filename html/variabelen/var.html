Variabelen

var

In ES 5 kan je ook het var keyword gebruiken voor de declaratie van een variabele.

Er wordt dan gebruik gemaakt van het sleutelwoord var om een constante te declareren

var variabeleNaam = initiële waarde

Er zijn een aantal nadelen aan verbonden aan het gebruik van var

  • var kan opnieuw gedeclareerd worden binnen dezelfde scope
  • Er is function scope. Dit is heel erg verschillend van block scope. Dit wil zeggen dat de variabele binnen de functie beschikbaar is (ook voor alle blokken in de functie). Als een var niet binnen een functie gedefinieerd is dan is krijgt deze globale scope (in een web applicatie is dit het window object)

Een aantal voorbeelden:

console.log("Gebruik van var");
var width;
console.log('width = ' + width);    // undefined
var height = 300;

function bar() {
    if (true) {
        var width = 200;
    }
    console.log('width in bar = ' + width); // 200
}

bar();
console.log('width = ' + width);    // undefined
console.log('height = ' + height);  // 300
// opnieuw declareren geeft geen syntaxfout
var width = 20;


console.log("Gebruik van let");
let width;
console.log('width = ' + width);    // undefined
let height = 300;
function bar() {
    if (true) {
        let width = 200;
    }
    console.log('width in bar = ' + width); // 200
}
bar();
console.log('width = ' + width);    // undefined
console.log('height = ' + height);  // 300

Best practice:

  • Gebruik altijd const
  • Gebruik let enkel als de variabele geüpdatet moet worden
  • Gebruik nooit var, tenzij voor compatibility issues met IE11

Oefening

  • Probeer eerst zelf te bedenken wat er in elk van de volgende gevallen zal worden weggeschreven naar de console
if (true) {
    var x = 1;
 }
 console.log(x);
 
 for (var i = 0; i < 10; i++ ) {
   // do stuff with i
 }
 console.log(i); 

if (true) {
    const x = 1;
 }
 console.log(x);

 for (let i = 0; i < 10; i++ ) {
   // do stuff with i
 }
 console.log(i);
